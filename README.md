# AeroGuard

航空守护是一个面向 Minecraft 1.21.1、NeoForge 和 Sable 1.2.2 的轻量级服务端补丁模组。

它主要用于缓解 Sable / 机械动力航空学大型载具在服务器中离开视距或模拟距离后，进入未加载状态并难以恢复显示的问题。

## 功能

- 只要载具中心区块已经加载，就允许 holding 状态的载具尝试恢复。
- 只有载具中心区块也未加载时，才允许运行中的载具进入 holding 状态。
- 边缘区块未加载时，不再直接导致整艘大型载具卸载。
- 不强制加载整艘载具周围所有区块。
- 不添加方块、物品或客户端界面。
- 玩家客户端不需要安装。

## 支持版本

- Minecraft: 1.21.1
- 加载器: NeoForge
- Sable: 1.2.2

当前版本只针对上述环境测试。其他 Minecraft、NeoForge 或 Sable 版本可能无法使用。

## 命令

```mcfunction
/aeroguard status
/aeroguard debug true
/aeroguard debug false
```

`/aeroguard status` 会显示模组是否启用、Sable 版本、恢复策略、卸载策略和调试日志状态。

## 安装

将构建出的 jar 放入服务端 `mods` 文件夹即可。

玩家客户端不需要安装本模组。

## 构建

```powershell
.\gradlew.bat build
```

构建产物位于：

```text
build/libs/
```

## 注意事项

航空守护不是区块加载器，也不会让所有载具永久运行。它只是减少大型载具因为边缘区块未加载而无法恢复的问题。

如果载具本身过大、结构异常、服务器性能过低，或者其他模组造成冲突，仍然可能出现问题。
